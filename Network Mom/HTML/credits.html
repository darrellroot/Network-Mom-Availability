Network Mom Availability Credits

Programmer

Network Mom Availability is written by Darrell Root for Network Mom LLC. Darrell worked as a Unix system administrator and network engineer for 25 years prior to moving into independent software development. Darrell holds a CCIE Emeritus certification (#8302). His goal is to make easy-to-use network management software for small and medium businesses.

Swift

Network Mom Availability is written in Swift: the new programming language developed by Apple. Chris Lattner led the original development. Swift is an excellent programming language. The strict types and optionals, in particular, prevent many programming errors. We also thank Apple for the entire Macintosh development environment.

Charts

Network Mom Availability uses the Charts framework available at https://cocoapods.org/pods/Charts by Daniel Cohen Gindi and Philipp Jahoda. Charts is made available under the Apache-2.0 license.

Swift-SMTP, LoggerAPI, BlueCryptor, BlueSocket, BlueSSLService

These libraries, made available on GitHub under https://github.com/IBM-Swift by IBM, are covered by the Apache-2.0 license and are used for sending email. Thank you IBM!

Audio Files

Audio files are from freesound.org and are available under the creative commons attribution license or creative commons 0 license. Audio files include sw_school_pa_alert.wav by jppi_Stu, aircraftalarm.wav by guitarguy1985, Doorbell Alert.wav by ChrisButler99, Warning sound by satchdev, Warning Signal Science Fiction SF by SooundsofTh.

SwiftyPing

SwiftyPing is available on Github at https://github.com/imas145/SwiftyPing by user imas145 under the MIT license. While we do not use the source code directly, figuring out how to send ICMP packets with Swift and the CFNetwork (Objective-C-based) framework was difficult. imas145's SwiftyPing example was essential.

MRTG/RRDtool

Tobi Oetiker's MRTG and RRDtool at https://www.oetiker.ch/en/oss/projects/ have been the gold standard for network bandwidth monitoring for decades. While we don't use any of their source code (wrong language!), our availability and latency data structures are designed similarly to Tobi's "round robin database". Their charts, in particular, are beautiful.
